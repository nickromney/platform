## Important Paths

| Path | Purpose |
| --- | --- |
| `kubernetes/kind` | Default Docker-backed platform runtime. |
| `kubernetes/lima` | VM-backed k3s runtime. |
| `kubernetes/slicer` | Optional Slicer microVM runtime. |
| `terraform/kubernetes` | Shared in-cluster platform resources. |
| `terraform/kubernetes/apps/gitea-actions-runner` | In-cluster Gitea Actions runner and review-environment RBAC. |
| `terraform/kubernetes/cluster-policies` | Cilium and Kyverno policy source. |
| `apps/backstage` | Portal UI, catalog, TechDocs, and scaffolding templates. |
| `apps/idp-core` | Product API for catalog, runtime, environments, deployments, secrets, and actions. |
| `apps/idp-sdk` | TypeScript client for the IDP API. |
| `apps/idp-mcp` | MCP wrapper over the IDP API. |
| `apps/platform-mcp` | Learning and diagnostic MCP server for platform workflows. |
| `apps/subnetcalc` | Main domain-rich sample app. |
| `apps/sentiment` | Smaller authenticated sample app. |
| `sites/docs` | Local Nextra documentation site. |
| `skills/use-platform` | Repo-local Codex skill and operator memory. |
| `docs/adr` | Design decisions. |
| `docs/ddd` | Domain language and context maps. |
| `scripts` | Root operator and validation helpers. |
| `tests` | Bats and workflow-level regression checks. |

## How To Navigate

Start from the focused Makefile for the thing you are operating. Read the
Terraform tree when the question is about in-cluster resources. Read the app
tree when the question is about sample workload behaviour.

<Callout type="warning">
  Avoid treating `docs/ddd` or `docs/adr` as generated reference. Those files
  contain design intent and should be read before renaming domain concepts.
</Callout>
