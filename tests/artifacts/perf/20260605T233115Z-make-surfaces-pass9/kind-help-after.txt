  make 100 apply
  make 900 plan
  make 920 plan
  make 900 apply AUTO_APPROVE=1
  make 920 apply AUTO_APPROVE=1

  make kind 100 apply
  make 900 check-security

Stages:
  100 - cluster available
  200 - cilium
  300 - hubble
  400 - argocd core
  500 - gitea + full argocd controllers
  600 - policies
  700 - app repos
  800 - observability
  900 - sso
  920 - langfuse

Stage examples:
  make 100 plan
  make 900 apply AUTO_APPROVE=1
  make 900 check-security

Default kubeconfigs:
  kind -> ~/.kube/kind-kind-local.yaml (split by default; merge explicitly into ~/.kube/config if needed)

Host runtime:
  macOS -> Docker Desktop
  Linux -> Docker Engine or Docker Desktop

Local helper modes:
  KIND_PRELOAD_IMAGES_MODE=auto|on|off                 -> preload images before apply (default: off, current behavior)
  MERGE_KUBECONFIG_TO_DEFAULT=1                        -> also merge ~/.kube/kind-kind-local.yaml into ~/.kube/config (default: 0)
  KIND_WORKER_COUNT=1|2|...                            -> worker node count (default: 1, so 2 total nodes)
  KIND_IMAGE_DISTRIBUTION_MODE=load|registry|hybrid|baked -> image distribution mode (default: registry)
  KIND_BAKED_NODE_IMAGE=<image-ref>                    -> required for KIND_IMAGE_DISTRIBUTION_MODE=hybrid|baked
  KIND_ENABLE_BACKSTAGE=auto|on|off                    -> Backstage portal gate (auto requires >=10GiB Docker memory)
  CHECK_VERSION_FORMAT=text|json                       -> render version audits as human text or machine-readable JSON (default: text)
  OIDC_RECOVERY_FORMAT=text|json                       -> render the OIDC recovery exercise as human text or machine-readable JSON (default: text)
  OIDC_RECOVERY_FORCE_MODE=nginx-rollout               -> controlled perturbation used to force the explicit recovery branch (default: nginx-rollout)
  PLATFORM_PROFILE_MODE=off|on|trace                   -> capture apply/plan timings and logs under <repo>/.run/profiles
