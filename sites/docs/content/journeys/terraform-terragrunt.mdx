# Learn Terraform And Terragrunt

Outcome: understand how Make, Terragrunt, OpenTofu or Terraform, tfvars, local
state, providers, Helm releases, and Kubernetes manifests divide
responsibility.

## Run

Always learn the IaC path with `plan` before `apply`:

```bash
cd ~/Developer/personal/platform
make -C kubernetes/kind 200 plan
make -C kubernetes/kind 200 apply AUTO_APPROVE=1
make -C kubernetes/kind 300 plan
```

To compare execution engines without changing the platform definition:

```bash
TG_TF_PATH=terraform make -C kubernetes/kind 300 plan
```

The default path is OpenTofu, selected by `terraform/root.hcl`.

## Inspect

Look at the input files first:

```bash
sed -n '1,120p' kubernetes/kind/stages/300-hubble.tfvars
sed -n '1,120p' kubernetes/kind/targets/kind.tfvars
sed -n '1,160p' terraform/root.hcl
```

Then inspect the generated state and backend files:

```bash
ls -la terraform/.run/kubernetes-kind
sed -n '1,80p' terraform/kubernetes/backend.tf
```

The core question is: which layer owns this behaviour?

| Layer | Owns |
| --- | --- |
| Make | Runtime checks, stage selection, image preloading, health checks. |
| Terragrunt | Local backend generation, tfvar wiring, execution-engine selection. |
| OpenTofu or Terraform | The graph of Kubernetes, Helm, local, TLS, random, and external resources. |
| Argo CD | Continuous reconciliation after the bootstrap handoff. |

## Try One Change

Do a read-only toggle exercise. Compare the plan for stage `800` with and
without a local override:

```hcl filename="local-platform.override.tfvars"
enable_loki = true
enable_tempo = true
```

```bash
PLATFORM_TFVARS=$PWD/local-platform.override.tfvars \
  make -C kubernetes/kind 800 plan
```

Delete the override when done. The lesson is that operator overrides should not
be committed into shared stage files unless the default platform path is
actually changing.

<Callout type="warning">
  Terragrunt cache directories are implementation detail. If a helper script
  assumes it is running from the source tree instead of the cache copy, it will
  fail in ways that look unrelated to Terraform.
</Callout>

### Acceptance Checks

```bash
make -C kubernetes/kind 800 plan
TG_TF_PATH=terraform make -C kubernetes/kind 800 plan
make -C kubernetes/kind 800 apply AUTO_APPROVE=1
```

The plan should show only the intended training diff. It should not recreate
the cluster, gateway, or unrelated Helm releases.
