# Source Repo Cleanup Plan

This is the follow-up plan for a separate agent working in
`~/Developer/personal/platform`. It is based on the tracked Markdown corpus and
the current Makefiles, stage tfvars, Terraform stack, shell scripts, and app
entrypoints.

## What Was Reviewed

The meaningful source-doc corpus is the tracked Markdown set in `platform`.
Raw filesystem discovery also finds `.venv`, `node_modules`, `.pytest_cache`,
Terragrunt cache, Playwright report, and test-result Markdown. Those generated
or dependency-owned files should not be treated as product documentation.

The durable implementation contract is:

- root `make` is informational
- focused Makefiles own mutating workflows
- Kind is the default Docker-backed reference path
- Lima reuse the shared Kubernetes stack after stage `100`
- stage tfvars define the enabled platform shape
- OpenTofu is the default Terraform-compatible engine through Terragrunt
- app trees are sample workloads, not the platform product itself

## Removal Candidates

| Candidate | Why remove or archive | Guardrail before deletion |
| --- | --- | --- |
| Retired Subnetcalc experiment docs | The previous Azure, C#, Python, and TypeScript variants were moved out of the platform repo so the app tree stays focused on the canonical Go workload. | Keep platform docs pointed at `apps/subnetcalc/app-go`; use the external archive branch for experiment history. |
| Docker Desktop managed Kubernetes target notes | `kubernetes/docker-desktop` is explicitly a research target, not a first-class runtime. | Keep only if those scripts are still used for regression research; otherwise archive the folder outside the main operator tree. |
| Observability wording | Default stage files set `enable_prometheus = true`, `enable_grafana = true`, and `enable_victoria_logs = true`. Retired observability variants were removed from active workflow surfaces. | Keep `README.md`, `kubernetes/kind/README.md`, `kubernetes/docker-desktop/FINDINGS.md`, and related notes aligned on the current stack. |
| Scattered version-management notes | Version pins, cooldowns, provider audits, and Kubernetes stack audits are documented in pieces. That makes the strongest free value of the repo harder to see. | Consolidate the source README and runtime READMEs around the same bump workflow described in the docs site: pin, cool, audit, bump one pathway, rebuild Kind, then promote. |
| Legacy APIM vendoring references | APIM simulator source now lives at `apps/apim-simulator`, so old `apps/subnetcalc/apim-simulator` and `apim-simulator.vendor.json` references point at removed paths. | Keep runtime wiring, DDD docs, and docs-site journeys on `apps/apim-simulator`; remove vendoring-script guidance. |
| Generated local state in working copies | `.venv`, `node_modules`, `.pytest_cache`, `.terragrunt-cache`, `dist`, Playwright reports, and test results show up in raw scans. | Confirm `.gitignore` covers each generated path; add missing ignores rather than deleting tracked source. |

## Implementation Order

1. **Build the inventory.** Run `git ls-files '*.md' '*.mdx'`, `rg` for each
   candidate path, and `git ls-files` for generated/dependency directories.
2. **Fix factual docs drift first.** Patch stage-800/900 observability wording
   so Prometheus/Grafana/VictoriaLogs are the implemented default and
   retired observability variants are not advertised as current pathways.
3. **Separate current docs from archives.** Keep only platform-current
   Subnetcalc docs in this repo and use the external experiment archive branch
   for retired implementation notes.
4. **Document the current observability pathway.** Keep Prometheus, Grafana,
   and VictoriaLogs as the active local stack story.
5. **Consolidate version-management docs.** Make the source repo explain how
   Kubernetes, K3s, Helm charts, images, providers, GitHub Actions, vendored
   code, and app dependencies move together without bypassing cooldown and
   pinning controls.
6. **Reconcile APIM integration.** Remove the old `apps/subnetcalc/apim-simulator`
   vendor story and keep shared compose, docs, and contracts pointed at
   `apps/apim-simulator`.
7. **Prune generated state.** Remove ignored local artifacts from the working
   tree and tighten ignores for anything that keeps reappearing.
8. **Run the smallest proof set.** Use `make`, `make lint`, `make fmt`, and
   targeted `make -C ... help/test` checks for any subtree touched by deletion.

## Non-Goals

Do not remove Kind, Lima, Lima, SD-WAN Lima, DDD notes, ADRs, policy audit
docs, or the app sample roots as part of the first pruning pass. Those are still
part of the source repo's current explanatory model.

<Callout type="warning">
  Treat deletion as a behaviour change when scripts, Make targets, Terraform
  inputs, or docs still reference the path. The first pass should make stale
  surfaces impossible to discover accidentally, not just make the tree smaller.
</Callout>
