# Subnetcalc

Subnetcalc is the main domain-rich sample app. It calculates IPv4 and IPv6
subnet details from a single Go service that can run as a backend, a frontend,
or both.

## Why It Exists

The platform needs a workload with enough shape to exercise routing, auth, API
mediation, frontend delivery, and network policy. Subnetcalc keeps those
concerns in one canonical Go app so the platform path is easier to verify.

## Hosting and Auth Patterns

| Pattern | Frontend | Backend | Auth model | Why it exists |
| --- | --- | --- | --- | --- |
| Anonymous local app | Embedded Go frontend | Go backend | none | Fastest domain and UI loop. |
| Split compose app | Go frontend role | Go backend role | none or OIDC/JWT | Models frontend-to-backend routing without a second codebase. |
| OIDC plus APIM | Protected Go frontend | Go backend behind APIM simulator | OIDC at proxy and token checks at API boundary | Mirrors the layered platform path. |

The important design choice is that runtime role, identity model, and platform
placement are separable while the implementation remains one app.

Keycloak is the Kubernetes stage-900 IdP, not a dependency of the subnet
calculation domain. The app supports anonymous and OIDC/JWT-protected API
calls. The integrated APIM simulator remains a separate supporting context
contract; the platform path configures it to validate Keycloak access tokens
with the `apim-simulator` audience.

## In the Kubernetes Stack

In the Kubernetes platform, Subnetcalc is reconciled through the shared
Terraform and GitOps path. Stage `700+` is where app repos and local workload
image flow become part of the platform story.

The platform path adds controls that compose does not fully model:

- Gateway API routing
- Keycloak and `oauth2-proxy` SSO at stage `900`
- Cilium namespace and project policies
- Kyverno label and image-source rules
- Gitea-seeded app repositories
- Argo CD reconciliation

## Common Mistakes

<Callout type="warning">
  APIM simulator source now lives at `apps/apim-simulator`. When changing
  APIM behaviour, update that subtree's contract matrix and focused tests
  alongside the platform path that consumes it.
</Callout>

- Run only one compose slice that owns the same ports.
- Keep app changes in `apps/subnetcalc/app-go`.
- Use the Kubernetes stack when you need GitOps, gateway, SSO, or policy
  behaviour.
