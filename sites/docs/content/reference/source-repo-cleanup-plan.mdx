# Source Repo Cleanup Plan

This is the follow-up plan for a separate agent working in
`~/Developer/personal/platform`. It is based on the tracked Markdown corpus and
the current Makefiles, stage tfvars, Terraform stack, shell scripts, and app
entrypoints.

## What Was Reviewed

The meaningful source-doc corpus is the tracked Markdown set in `platform`.
Raw filesystem discovery also finds `.venv`, `node_modules`, `.pytest_cache`,
Terragrunt cache, Playwright report, and test-result Markdown. Those generated
or dependency-owned files should not be treated as product documentation.

The durable implementation contract is:

- root `make` is informational
- focused Makefiles own mutating workflows
- Kind is the default Docker-backed reference path
- Lima and Slicer reuse the shared Kubernetes stack after stage `100`
- stage tfvars define the enabled platform shape
- OpenTofu is the default Terraform-compatible engine through Terragrunt
- app trees are sample workloads, not the platform product itself

## Implementation Order

1. **Build the inventory.** Run `git ls-files '*.md' '*.mdx'`, `rg` for each
   candidate path, and `git ls-files` for generated/dependency directories.
2. **Fix factual docs drift first.** Patch stage-800/900 observability wording
   so Prometheus/Grafana/VictoriaLogs are the implemented default and
   Loki/SigNoz are documented as heavier alternatives.
3. **Separate current docs from archives.** In `apps/subnetcalc/docs`, keep
   active runbooks near code and move dated plans/session summaries under an
   explicit archive folder, or delete them if no current reference remains.
4. **Document the optional observability pathways.** Keep SigNoz and Loki
   code, scripts, and chart references if they still work, but make their
   memory cost and non-default status explicit.
5. **Consolidate version-management docs.** Make the source repo explain how
   Kubernetes, K3s, Helm charts, images, providers, GitHub Actions, vendored
   code, and app dependencies move together without bypassing cooldown and
   pinning controls.
6. **Reconcile APIM vendoring.** Make `apim-simulator.vendor.json`, the
   vendoring script, and the tracked vendored subtree agree.
7. **Prune generated state.** Remove ignored local artifacts from the working
   tree and tighten ignores for anything that keeps reappearing.
8. **Run the smallest proof set.** Use `make`, `make lint`, `make fmt`, and
   targeted `make -C ... help/test` checks for any subtree touched by deletion.

## Non-Goals

Do not remove Kind, Lima, Slicer, SD-WAN Lima, DDD notes, ADRs, policy audit
docs, or the app sample roots as part of the first pruning pass. Those are still
part of the source repo's current explanatory model.

<Callout type="warning">
  Treat deletion as a behaviour change when scripts, Make targets, Terraform
  inputs, or docs still reference the path. The first pass should make stale
  surfaces impossible to discover accidentally, not just make the tree smaller.
</Callout>
