# Learn GitOps With Argo CD

Outcome: understand the handoff from Terraform bootstrap to Argo CD
reconciliation: Terraform installs Argo CD and creates selected `Application`
objects; Argo CD then syncs charts and manifests from the in-cluster Gitea
repository.

## Run

```bash
cd ~/Developer/personal/platform
make -C kubernetes/kind 400 apply AUTO_APPROVE=1
kubectl -n argocd get deploy,svc

make -C kubernetes/kind 500 apply AUTO_APPROVE=1
kubectl -n argocd get applications.argoproj.io
```

Then add policies and app repos:

```bash
make -C kubernetes/kind 700 apply AUTO_APPROVE=1
kubectl -n argocd get app -o wide
```

## Inspect

Use Argo CD objects as the map:

```bash
kubectl -n argocd get app
kubectl -n argocd describe app kyverno-policies
kubectl -n argocd describe app dev
kubectl -n argocd get secret | rg "repo|gitea"
```

Check which manifests Argo CD is expected to sync:

```bash
find terraform/kubernetes/apps/argocd-apps -maxdepth 1 -type f | sort
find terraform/kubernetes/apps/dev terraform/kubernetes/apps/uat -maxdepth 2 -type f
```

The platform currently uses direct Terraform-created Argo CD Applications by
default. The app-of-apps directory exists as an optional path, but stage `900`
sets `enable_app_of_apps = false`.

## Try One Change

Pick one Argo CD Application and trace it from desired state to live state:

```bash
kubectl -n argocd get app dev -o yaml | yq '.spec.source.path, .spec.destination.namespace'
kubectl -n dev get deploy,svc
```

Then inspect the source path:

```bash
sed -n '1,120p' terraform/kubernetes/apps/dev/kustomization.yaml
sed -n '1,80p' terraform/kubernetes/apps/workloads/base/kustomization.yaml
```

The lesson is that GitOps is a feedback loop. A commit or generated repo sync
changes desired state; Argo CD reconciles live cluster state to match.

<Callout type="warning">
  Do not confuse "Argo CD is installed" with "GitOps is in charge of
  everything." Bootstrap resources, local credentials, repo seeding, and some
  Application objects are still Terraform-managed in this platform.
</Callout>

## Closure Slice

Use `hello-platform` as the promotion example. It now has separate dev and UAT
image patch files:

| Artifact | Purpose |
| --- | --- |
| [`terraform/kubernetes/apps/dev/hello-platform-image-patch.yaml`](https://github.com/nickromney/platform/blob/main/terraform/kubernetes/apps/dev/hello-platform-image-patch.yaml) | Dev image reference. |
| [`terraform/kubernetes/apps/uat/hello-platform-image-patch.yaml`](https://github.com/nickromney/platform/blob/main/terraform/kubernetes/apps/uat/hello-platform-image-patch.yaml) | UAT image reference. |
| [`terraform/kubernetes/apps/dev/kustomization.yaml`](https://github.com/nickromney/platform/blob/main/terraform/kubernetes/apps/dev/kustomization.yaml) | Includes the dev promotion patch. |
| [`terraform/kubernetes/apps/uat/kustomization.yaml`](https://github.com/nickromney/platform/blob/main/terraform/kubernetes/apps/uat/kustomization.yaml) | Includes the UAT promotion patch. |

The dev commit proves the new version. The UAT commit proves promotion. Both
are visible in Gitea and in Argo CD history once synced.

### Acceptance Checks

```bash
kubectl -n argocd get app dev uat -o wide
kubectl -n dev get deploy hello-platform -o jsonpath='{.spec.template.spec.containers[0].image}'
kubectl -n uat get deploy hello-platform -o jsonpath='{.spec.template.spec.containers[0].image}'
```

Finish by reverting the UAT promotion commit and observing Argo CD return UAT
to the previous image.
